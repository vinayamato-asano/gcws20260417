# データモデル仕様

本アプリは現状、サーバー側でデータモデルやDBスキーマを持ちません。

- タイマー状態・完了回数・集中時間などは、すべてフロントエンド（JavaScript）で管理されます。
- サーバー側（Flask）はHTMLテンプレートの返却のみを担当し、データ保存やユーザー管理は行いません。

---

# 今後の拡張例
- セッション管理やユーザーごとの進捗保存を行う場合、以下のようなモデルが想定されます：
  - User（ユーザー情報）
  - PomodoroSession（1回のタイマー記録）
  - DailyStats（日ごとの統計）
